# CoopAndreas v0.2.11 ‚Äî Handoff Notes

## What Changed in v0.2.11

### 1. Stats Separation (PERSONAL vs SHARED)

**File**: `client/src/CStatsSync.cpp`

`ApplyProgressSnapshot()` no longer does blind `memcpy` of all stats. Instead:

**PERSONAL float stats** (NOT overwritten by remote player):
- Body: Fat(21), Stamina(22), Muscle(23), MaxHealth(24), SexAppeal(25)
- Weapons: Pistol(69), SilencedPistol(70), DesertEagle(71), Shotgun(72), SawnOff(73), CombatShotgun(74), Uzi(75), MP5(76), AK47(77), M4(78), Sniper(79)
- Other: SexAppealClothes(80), Gambling(81)

**PERSONAL int stats** (array offset = stat_id - 120):
- DrivingSkill(40), FlyingSkill(103), LungCapacity(105), BikeSkill(109), CycleSkill(110)

**SHARED int stats** (MAX-merged ‚Äî both players get highest value):
- TaxiMoneyMade(29), FiresExtinguished(32), VigilanteLevel(37), ParamedicLevel(38), FirefighterLevel(39)

All other stats: overwritten by remote (mission progress, etc.)

### 2. Wanted Level ‚Äî MAX Sync

**Files**: `client/src/CStatsSync.cpp`, `client/src/CPacketHandler.cpp`

Both `WantedLevelSync__Handle` and `ApplyProgressSnapshot` now use MAX logic:
- `maxLevel = max(localLevel, remoteLevel)`
- If remote player has higher wanted level, local player gets upgraded to match
- Both players always share the highest wanted level

### 3. Money ‚Äî Fully Separate

**File**: `client/src/CPacketHandler.cpp`

- `MoneySync__Handle`: No longer overwrites local money when receiving own playerid
- `ApplyProgressSnapshot`: Removed money overwrite entirely
- Each player earns and spends independently
- Remote player's money still stored on `CNetworkPlayer::m_nMoney` for display

### 4. Properties ‚Äî Fully Separate

**File**: `client/src/CStatsSync.cpp`

- `ApplyProgressSnapshot`: No longer copies `ownedProperties` from remote
- No longer calls `PushOwnedPropertiesToScm()` with remote data
- Each player buys properties independently
- Remote player's properties still stored on `CNetworkPlayer::m_ownedProperties` for reference

### 5. Side Missions ‚Äî Shared via MAX Merge

Ambulance, Vigilante, Firefighter, Taxi progress uses MAX merge in `ApplyProgressSnapshot`:
- If Player A completes Vigilante level 12 and Player B has level 8, both get level 12
- Tracked via int stat offsets in `kSharedMaxIntStatOffsets[]`

## Architecture Notes

- **Host/Client model**: Host runs SCM scripts, sends opcodes via `COpCodeSync`
- **Stats transmission**: `PLAYER_STATS` packet contains ALL stats ‚Äî filtering is done on RECEIVE side
- **Stat arrays**: `CStats::StatTypesFloat[83]` (indices 0-82), `CStats::StatTypesInt[224]` (stat IDs 120-343, array offset = id-120)
- **Properties**: 32 slots at SCM global base 728. `PullOwnedPropertiesFromScm()` reads them each frame
- **Schools**: Still fully shared (schoolProgress, schoolMedals)
